# Clipboard Hub 当前代码状态核验

> 核验日期：2026-06-23  
> 范围：`src/`、`tests/`、README、requirements 与现有设计文档  
> 测试结果：14 passed  

## 一、结论

现有代码已经修复初始审查中的部分 Critical 问题，但尚未完整达到 V1 设计文档标记的“已实施”状态。原初审文档反映的是更早版本，也不能直接代表当前代码。

## 二、已经实现

- 四个核心模块已经存在并完成基本连接。
- 文本、HTML、图片的剪贴板轮询框架已经存在。
- Store 支持新增、连续去重、删除、搜索和按 ID 查询。
- 14 个现有单元测试全部通过。
- TabBar 已改为 `380×16px` 半透明玻璃条。
- TabBar 支持拖动、面板跟随和延时动画吸附。
- HistoryPanel 已实现透明玻璃绘制。
- 每条记录已经使用独立卡片，并具备删除按钮和确认框。
- 搜索无结果时显示“无匹配记录”。
- 点击历史记录后面板不会自动隐藏。
- 自复制接口已经传入 MD5。
- 系统托盘和退出菜单已经实现。

## 三、尚未实现或实现不完整

### 高优先级

1. 历史仅保存在内存，退出后全部丢失。
2. HTML 重新复制仍调用 `clipboard.setText()`，富文本格式丢失。
3. Watcher 在图片、HTML、文本之间只选择第一种格式，不能保存同一次复制的全部 MIME 数据。
4. `app_source` 始终为 `unknown`。
5. 没有复制成功或失败的视觉反馈。

### 中优先级

1. TabBar 使用 `QApplication.primaryScreen()`，不能吸附到其当前所在显示器。
2. TabBar 和面板没有完整的多显示器越界处理。
3. 搜索框未启用清空按钮。
4. 图片继续以 base64 字符串保存在内存中，体积大于原始 PNG。
5. 没有历史条数和总容量限制。
6. HTML 预览直接截断源码，可能显示破碎标签。
7. `delete()` 即使 ID 不存在也会发射 `items_changed`。
8. 当前自复制逻辑无条件跳过下一轮轮询；如果用户在该轮前立即复制了别的内容，可能漏记。

### 测试缺口

- 没有真实或 mock MIME 数据的 Watcher 测试。
- 没有 HTML、图片和多格式写回测试。
- 没有持久化、容量淘汰和重启恢复测试。
- 没有多显示器几何计算测试。
- 没有来源应用测试。
- 没有复制反馈状态测试。
- 没有删除信号、空内容和异常降级的完整测试。

## 四、文档一致性问题

- V1 设计文档此前标记“全部代码实现完成”和“Critical 问题修复”，但实际仍存在 HTML 富文本恢复等缺口。
- 实施计划仍全部为未勾选状态，且示例代码是早期深色主题版本。
- 初始审查中的“删除按钮缺失”和“无退出机制”已经过时；HTML 写回问题仍然存在。
- V1 设计与当前代码都没有持久化方案，现已由 V2 设计取代。
