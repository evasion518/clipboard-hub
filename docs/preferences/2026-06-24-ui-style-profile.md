# Clipboard Hub UI 风格档案

> 创建日期：2026-06-24  
> 用途：记录本项目在界面设计、审美取向、实现边界上的长期偏好，供后续继续迭代时直接复用。

## 一、审美结论

当前项目的 UI 方向不是“纯功能后台”，也不是“强装饰概念稿”，而是：

- 偏 Windows / Fluent 的桌面工具
- 浅色为主，但保留系统主题跟随能力
- 重点追求“高级感、层次感、成品感”
- 风格关键词：香槟暖灰、柔光、轻奢、舒展、克制

一句话总结：

> 做成“浅色 Fluent 卡片工具”，看起来像一款被认真打磨过的桌面成品，而不是默认控件拼装界面。

## 二、用户明确表达过的偏好

### 2.1 视觉风格

用户在多轮选择中明确确认：

- 选 Fluent 卡片风，而不是极简工具风或纯原生 Windows 风
- 浅色为主，而不是深色为主
- 重点偏“更高级”，不是单纯“更清爽”
- 历史项早期走“舒展型”，当前最终口径已收敛为紧凑文本复制框
- 颜色强调偏“香槟暖灰”，不是微蓝银灰或蓝紫高光

### 2.2 精修方向

在进一步美化阶段，用户又明确确认：

- 图片卡片方案已取消；当前不保存、不预览、不写回图片
- 细腻感和版式层级都要，但优先做“细腻感”
- 接受“柔光成品版”作为整体 polish 方向

## 三、设计原则

后续新增或修改 UI 时，优先遵守下面这些原则。

### 3.1 观感原则

- 舒展，但不松散
- 精致，但不花哨
- 有层次，但不杂乱
- 高级感来自留白、边框、阴影、高光和比例，而不是堆效果

### 3.2 文案原则

- 全部可见文案统一中文
- 语气简洁、自然、偏系统工具
- 不卖萌，不使用互联网口语，不做营销式表达

示例风格：

- `搜索剪贴板历史`
- `还没有剪贴板记录`
- `没有找到匹配内容`
- `已复制`
- `复制失败`
- `确认删除`

### 3.3 反馈原则

- 成功/失败都要明确，但不要刺眼
- 成功反馈更像“被点亮”，不是强提示框
- 失败反馈也要保持界面气质，不要突然变成生硬警告色块

## 四、组件级偏好

### 4.1 HistoryPanel

- 面板应继续保持成品化卡片风
- 固定尺寸可以接受，但视觉上要像“竖向内容页”
- 搜索区、卡片区、空状态区要拉开节奏，避免挤

### 4.2 SearchBox

- 更像一块浅香槟胶囊，而不是普通输入框
- 聚焦态应靠柔和高光和边框变化体现
- 占位文字颜色必须显式控制，避免依赖平台默认

### 4.3 历史复制框

- 历史项更像紧凑复制框，而不是大内容卡片
- 左侧显示倒序序号
- 中部显示正文，最多两行，超出部分用渐隐遮罩隐藏
- 右侧是等高删除覆盖条，宽度为高度的一半
- 默认不展示类型标签、来源应用或窗口标题
- 文件复制显示为本地路径文本
- 图片不保存、不预览、不写回

### 4.4 图片与文件

当前最终口径是：

- 图片剪贴板不再生成图片历史项
- 旧记录里即使保留图片字段，也不作为图片写回
- 文件剪贴板转为换行分隔的本地路径文本
- 历史列表始终优先服务文本浏览和纯文本复制

### 4.5 TabBar

- 不能只“能用”，也要和面板风格统一
- 允许比最初版本更舒展一点
- 但仍应保持“悬浮入口条”的克制感，不做成厚重按钮

## 五、实现边界

后续做 UI 优化时，优先遵守这些边界：

- 优先改视觉，不改业务规则
- 优先改主题、布局、文案、反馈，不改数据结构
- 对于已稳定的剪贴板捕获/写回/持久化逻辑，默认不碰
- UI 改动尽量伴随测试更新，避免只改外观不补验证

## 六、当前实现与偏好关系

截至 2026-06-24，当前代码已经落下这些方向：

- 中文文案统一
- 冷白微蓝的 iOS 玻璃主题
- 更紧凑的复制框布局和圆角
- 柔和阴影和反馈态
- 倒序序号、两行渐隐、柔和渐变分割线
- 等高删除覆盖条
- TabBar 与面板主题统一

仍可继续抬升的方向：

- 反馈态继续保持精致状态徽标
- 元信息排版更高级
- 颜色 token 独立成更清晰的设计系统
- 若继续做视觉迭代，优先加深“细腻感”，而不是增加复杂度

## 七、下次继续时的默认做法

如果下次继续 UI 迭代，默认按下面顺序思考：

1. 先确认是否仍保持“冷白玻璃 + 紧凑复制框 + 内容优先”
2. 优先看是否能提升层次、留白、反馈质感
3. 新增视觉时先问：会不会破坏工具感？
4. 优先用小而精的变化提升成品感，不做大幅重构
5. 文案默认继续保持当前中文系统工具风格

## 八、2026-06-24 手动验收补充偏好

本轮手动体验后，新增以下明确偏好，后续默认直接沿用：

- 搜索框输入属于“正在交互”，不能因为悬浮状态切换而自动收回面板
- 历史卡片默认不展示“文本 / 网页 / 图片”这类类型标签，避免视觉噪音
- 历史卡片默认不展示复制来源或窗口标题，列表优先突出内容本身
- 每条历史记录之间必须有清晰边界，第一眼就能快速分辨条目分隔
- 点击复制后显示 `已复制`，鼠标移出面板后正常收回

一句话总结：

> 这个界面后续继续走“内容优先、标签减量、边界清楚、复制后可收回”的方向。

## 九、2026-06-24 iOS 透明玻璃版视觉方案

本轮又新增一条更明确的视觉方向：在不改当前交互结构的前提下，把主界面材质从“暖灰 Fluent 卡片”推进到“iOS 透明玻璃卡片”。

### 9.1 覆盖范围

本次玻璃感改造只覆盖以下区域：

- 外层主面板
- 搜索框
- 每条历史卡片

默认不改：

- TabBar 的交互逻辑
- 列表结构
- 搜索逻辑
- 删除/复制行为
- 数据结构与业务规则

### 9.2 目标质感

目标不是做夸张霓虹或高饱和科技风，而是做偏 iOS 的轻玻璃感：

- 偏白、偏冷、轻微带一点浅蓝
- 半透明乳白底，而不是实心灰块
- 用柔和描边、高光和轻阴影表现层次
- 保持整洁、克制、成品感

一句话概括：

> 保留现有布局骨架，把材质语言改成“偏白微蓝的 iOS 透明玻璃”。

### 9.3 分组件要求

#### HistoryPanel

- 主面板改为乳白半透明玻璃背景
- 外圈描边改为冷白到浅蓝的轻渐变感
- 圆角继续保持大圆角
- 阴影减重、变软，避免厚重卡片感

#### SearchBox

- 搜索框改成悬浮玻璃胶囊
- 聚焦时通过边缘提亮体现交互，而不是明显实心换底
- 占位文字继续保持清晰可读，但整体观感要更轻

#### HistoryCard

- 每条历史记录改成独立玻璃复制框
- 边界主要依靠透明底、细描边、柔阴影和留白拉开
- 保持“内容优先”，不要重新加回类型标签和来源标签
- 时间与反馈徽标也要跟随变成更轻的玻璃徽标感
- 内容最多两行，超出时用渐隐遮罩隐藏
- 复制框宽度略短于搜索框，避免横向滚动条

### 9.4 可读性边界

虽然走玻璃风，但必须守住下面几条边界：

- 文字仍然优先深色高对比，不能发灰发虚
- 透明度不能高到影响历史内容的第一眼识别
- 描边不能做成廉价霓虹蓝
- 阴影不能重到像安卓厚卡片

### 9.5 默认实现策略

后续如果继续按这条视觉方向迭代，默认优先顺序是：

1. 先改颜色 token，让整体从暖灰转向冷白微蓝
2. 再改面板、搜索框、卡片的背景/描边/阴影
3. 最后统一时间徽标、反馈徽标的玻璃化细节

默认判断标准：

- 第一眼像更轻、更透、更精致
- 第二眼仍然清楚、稳定、好读
- 看起来像系统级工具，而不是概念稿

### 9.6 参考图布局约束

后续继续做这个界面时，布局默认以“上搜索、下列表、单行条目”的参考图为准：

- 搜索框单独位于顶部头部区域
- 搜索区下方有一条明确横向分隔线
- 历史项默认采用紧凑复制框，而不是大块内容卡片
- 每条右侧使用等高删除覆盖条，不再使用文字型删除按钮
- 条目之间使用更薄的渐变分割线，增强第一眼扫描感

### 9.7 最新验收反馈与已落实调整

这一轮用户又进一步明确了“参考图只参考布局，不参考颜色”，因此默认口径更新为：

- 布局向参考图靠拢
- 颜色由当前实现自主设计，优先做得更高级、更克制
- 不追随参考图里的朴素灰色或直接照抄配色

当前已经落实的细化项：

- 主配色切到冷白、雾蓝、青灰高光的 iOS 玻璃方向
- 右侧删除按钮改成等高玻璃覆盖条，宽度为高度的二分之一
- 历史条目之间的垂直间距已明显收紧
- 每个条目的上边距与下边距已调整为对称
- 条目间分隔线已改为更薄、更融入面板的渐变线
- 复制框固定宽度略短于搜索框，并关闭横向滚动条
- 复制后显示 `已复制`，鼠标移出面板正常收回

后续默认继续沿用：

- “布局听参考图，颜色走高级玻璃感”
- “条目紧凑、分隔清晰、扫描感强、复制后可自然收回”

## 十、2026-06-24 最终问题修复口径

本轮最终确认并落实的产品行为如下：

- 面板内点击复制不会重新生成一条同内容历史。
- 重新复制只写纯文本；HTML 剥离标签后写回，图片不写回。
- 图片剪贴板不保存、不预览；文件剪贴板显示本地路径文本。
- 点击复制后复制框变色并显示 `已复制`。
- 搜索输入期间面板保持展开；复制成功后鼠标移出面板正常收回。
- 复制框上下边距一致，高度更低，内容左对齐并垂直居中。
- 内容最多两行，超出部分用渐隐遮罩淡化隐藏。
- 每条内容显示倒序序号。
- 分割线高度更小，并使用与玻璃面板融合的渐变线。
- 复制框略短于搜索框，列表不出现横向滚动条。
- 删除按钮是复制框右侧等高覆盖条，宽度为高度的一半。

## 十一、后续产品化改进建议

如果后续继续推进 Clipboard Hub，默认优先做“稳定、隐私、可达性、可维护”的小步产品化，而不是一次性加入大而复杂的功能。

当前已完成：

1. 数据路径与应用身份修正
   - 显式设置 Qt application name / organization name。
   - 确保数据库稳定落在 `Clipboard Hub` 应用目录。
   - 避免路径漂移造成用户误以为历史丢失。

后续推荐优先级如下：

1. 设置入口与隐私开关
   - 先从托盘菜单提供轻量设置入口。
   - 优先提供“是否记录来源应用和窗口标题”开关。
   - 再考虑最大历史数量、主题模式、开机启动。
2. 全局快捷键
   - 可选支持 `Ctrl+Shift+V` 这类快捷键打开面板。
   - 必须可配置或可关闭，避免和系统、输入法、常用软件冲突。
3. 批量管理
   - “清空历史 + 确认 + 刷新面板”已落地为一键删除按钮。
   - 暂不急着做复杂批量选择。
4. 搜索体验增强
   - 可增加匹配词高亮。
   - 可明确搜索结果序号语义。
   - 可让搜索框聚焦时更容易替换关键词。
5. 操作区可发现性
   - 删除按钮可增加克制的 `x` 或垃圾桶图标。
   - 增加 tooltip 和可访问名称。
   - 仍需保持列表内容优先，不让图标喧宾夺主。
6. 手动验收与截图记录
   - 将浏览器、Office、记事本、文件复制、多显示器、主题、重启恢复等真实体验写入 `docs/reviews/`。
   - 截图可作为后续 UI 调整的视觉基线。

一句话总结：

> 下一阶段优先把它做成“可长期自己用、路径稳定、隐私可控、打开方便、清理容易”的桌面小工具。
